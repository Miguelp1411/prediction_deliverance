Cambios aplicados en esta versión parcheada

1) Selección de mejor modelo
- OccurrenceModel ahora monitoriza weekly_total_mae.
- TemporalModel ahora monitoriza e2e_compound_score.
- El evaluador extra temporal se ejecuta en cada epoch para que el monitor sea consistente.

2) Señal estacional explícita
- Occurrence: se añaden lags 4/26/52 semanas como features directas.
- Occurrence: se combina una baseline estacional con logits neuronales (baseline + residual neural).
- Temporal: se añaden como features directas el start_bin de lag 26, lag 52 y la mediana histórica por tarea/ocurrencia.

3) Compatibilidad
- FEATURE_SCHEMA_VERSION sube a 4.
- Los checkpoints antiguos no son compatibles y deben regenerarse con train.py.
